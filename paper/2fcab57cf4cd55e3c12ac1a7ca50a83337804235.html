The Transformer is a deep learning architecture introduced in 2017 that relies entirely on self-attention mechanisms for sequence transduction, eschewing the recurrent and convolutional layers previously standard in the field. It established a new state-of-the-art in machine translation, achieving 28.4 BLEU on the WMT 2014 English-to-German task while requiring significantly less training time due to its highly parallelizable nature. By reducing the maximum path length between distant positions to a constant number of operations, the Transformer effectively addresses the long-range dependency problem inherent in sequential models.

The Transformer Architecture: An Attention-Only Design

Figure 1: The Transformer - model architecture.

The Transformer follows the traditional encoder-decoder structure but replaces recurrent and convolutional sub-layers with stacked self-attention. As shown in Figure 1, the encoder consists of a stack of six identical layers, each containing two sub-layers: a multi-head self-attention mechanism and a position-wise fully connected feed-forward network. Around each sub-layer, the model employs a residual connection followed by layer normalization, ensuring that gradients can flow effectively through the deep stack.

The decoder also employs a stack of six layers but introduces a third sub-layer that performs multi-head attention over the encoder's output. To maintain the auto-regressive property of sequence generation, the decoder's self-attention sub-layers are modified with masking. This prevents any given position from 'looking ahead' at subsequent tokens during the training process, ensuring that the prediction for position i depends only on the known outputs at positions less than i.

Scaled Dot-Product Attention Mechanism

Figure 2: (left) Scaled Dot-Product Attention. (right) Multi-Head Attention consists of several

The fundamental building block of the model is Scaled Dot-Product Attention, illustrated in Figure 2 (left). This mechanism maps a query (Q) and a set of key-value pairs (K, V) to an output. Unlike standard dot-product attention, this implementation introduces a scaling factor of 1/√d_k. This scaling is critical for performance when the dimensionality d_k is large, as it prevents the dot products from growing to magnitudes that would push the softmax function into regions with extremely small gradients.

Technically, the output is computed as a weighted sum of the values, where the weight assigned to each value is determined by the compatibility (dot product) of the query with its corresponding key. This operation can be efficiently implemented using highly optimized matrix multiplication code, making it significantly faster and more space-efficient than additive attention mechanisms used in earlier recurrent models.

Multi-Head Attention and Subspace Parallelism

Figure 2: (left) Scaled Dot-Product Attention. (right) Multi-Head Attention consists of several

To improve the model's ability to focus on different parts of the sequence simultaneously, the Transformer employs Multi-Head Attention, as shown in Figure 2 (right). Instead of performing a single attention function with d_model-dimensional vectors, the model linearly projects the queries, keys, and values h times into lower-dimensional subspaces (d_k, d_k, d_v). The attention function is then executed in parallel across these heads.

This design allows the model to jointly attend to information from different representation subspaces at different positions. For example, one head might focus on syntactic dependencies while another focuses on semantic relationships. The outputs from each head are concatenated and projected once more to produce the final d_model-dimensional representation. In the base model, h=8 heads are used, each with a dimensionality of 64, keeping the total computational cost equivalent to single-head attention at full dimensionality.

Position-wise Feed-Forward Networks

In addition to attention sub-layers, each layer in both the encoder and decoder contains a position-wise fully connected feed-forward network (FFN). This FFN is applied to each position separately and identically. It consists of two linear transformations with a ReLU activation in between. While the linear transformations are consistent across different positions within a layer, they use distinct parameters from layer to layer.

The internal dimensionality of this FFN (d_ff) is typically larger than the model dimensionality; in the base architecture, d_ff = 2048 while d_model = 512. This component can be viewed as two convolutions with kernel size 1, providing the necessary non-linearity and capacity to process the features extracted by the attention mechanisms.

Sinusoidal Positional Encoding

Because the Transformer lacks recurrence or convolution, it possesses no inherent knowledge of the relative or absolute positions of tokens in a sequence. To address this, 'positional encodings' are added to the input embeddings at the base of the encoder and decoder stacks. These encodings have the same dimension as the embeddings, allowing the two to be summed directly.

The researchers chose a fixed sinusoidal encoding using sine and cosine functions of different frequencies. This choice was motivated by the hypothesis that it would allow the model to easily learn to attend by relative positions, as for any fixed offset k, the encoding at position pos+k can be represented as a linear function of the encoding at position pos. Furthermore, this periodic representation allows the model to potentially extrapolate to sequence lengths longer than those encountered during training.

Concept Breakdown

Self-Attention

An attention mechanism relating different positions of a single sequence to compute a representation of that sequence. It allows for O(1) path length between any two tokens.

Encoder-Decoder Attention

A specific sub-layer in the decoder where queries come from the previous decoder layer and keys/values come from the encoder output, enabling the decoder to focus on relevant parts of the input.

Residual Connections

Shortcuts that bypass one or more layers, implemented as LayerNorm(x + Sublayer(x)), which help mitigate the vanishing gradient problem in deep networks.

Label Smoothing

A regularization technique that prevents the model from becoming too confident in its predictions, improving accuracy and bleu score at the cost of perplexity.

Mathematics

Scaled Dot-Product Attention

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
Calculates the attention weights by taking the dot product of queries and keys, scaling by the square root of the key dimension to stabilize gradients, applying softmax, and using the result to weight the values.
SymbolMeaning
$Q$Matrix of query vectors representing the current tokens
$K$Matrix of key vectors representing the tokens being attended to
$V$Matrix of value vectors containing the information to be extracted
$d_k$Dimensionality of the key and query vectors

Multi-Head Attention

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, ..., \text{head}_h)W^O$$
Runs multiple attention 'heads' in parallel across different linear projections of the input, then concatenates and projects the result.
SymbolMeaning
$h$Number of parallel attention layers (heads)
$W^O$Learned parameter matrix for the final linear projection
$\text{head}_i$\text{Attention}(QW^Q_i, KW^K_i, VW^V_i)

Position-wise Feed-Forward Network

$$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$$
A two-layer linear transformation with a ReLU activation applied to each token position independently.
SymbolMeaning
$W_1, W_2$Learned weight matrices for the two linear layers
$b_1, b_2$Bias vectors for the two linear layers

Sinusoidal Positional Encoding

$$PE_{(pos, 2i)} = \sin(pos/10000^{2i/d_{model}})$$
Generates a unique position vector based on sine and cosine waves of varying frequencies, allowing the model to distinguish token order.
SymbolMeaning
$pos$The position of the token in the sequence
$i$The dimension index within the encoding vector
$d_{model}$The total dimensionality of the model embeddings
