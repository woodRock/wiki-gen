Mamba is a breakthrough sequence modeling architecture that introduces selective state space models (SSMs) to achieve linear-time scaling and high performance across modalities including language, genomics, and audio. By allowing SSM parameters to be functions of the input, Mamba enables content-based reasoning—a key limitation of previous subquadratic architectures. Combined with a hardware-aware parallel algorithm, Mamba matches or exceeds Transformer performance while offering five times higher throughput and the ability to process million-length sequences.

From Linear SSMs to Selective Selection

Traditional State Space Models (SSMs) are linear time-invariant (LTI) systems that map an input sequence to an output sequence through an intermediate latent state. While efficient due to their convolutional representation, LTI systems struggle with content-based reasoning because their transition matrices are fixed and independent of the input.

Mamba introduces 'Selection' by making the SSM parameters ($$B$$, $$C$$, and $$\Delta$$) functions of the input $$x$$. This allows the model to 'selectively' propagate or forget information based on the current token, enabling it to handle discrete modalities like language where specific words (such as 'if' or 'else') dictate the importance of subsequent information.

Hardware-Aware Parallel Scan

The introduction of input dependency breaks the convolutional property of SSMs, which would typically require a slow recurrent computation. Mamba overcomes this through a hardware-aware parallel scan algorithm. Instead of materializing the full expanded state in slow GPU high-bandwidth memory (HBM), the model performs the discretization and recurrence in fast static random-access memory (SRAM).

By utilizing the memory hierarchy of modern GPUs, Mamba achieves the efficiency of convolutions while maintaining the flexibility of recurrent models. This allows the model to scale linearly with sequence length, making it practical for processing extremely long contexts that would be computationally prohibitive for Transformers' $$O(L^2)$$ attention.

The Mamba Block Architecture

The Mamba architecture simplifies the standard deep learning stack by integrating the selective SSM into a streamlined block. Unlike Transformers, which alternate between attention and multi-layer perceptron (MLP) layers, Mamba uses a single homogeneous block that combines linear projections, 1D convolutions, and the selective SSM.

This design removes the need for attention mechanisms entirely, significantly reducing the memory footprint for key-value (KV) caches during inference. The result is a model that is both faster to train and more efficient to deploy, particularly for long-generation tasks.

Empirical Performance and Scaling

In language modeling benchmarks, Mamba-3B matches the performance of Transformers twice its size. It demonstrates consistent improvements as the sequence length increases, whereas Transformers often suffer from performance degradation or memory exhaustion beyond their training window.

Beyond language, Mamba excels in genomics (long DNA sequences) and audio modeling, where its ability to handle millions of time steps with linear complexity provides a decisive advantage over previous state-of-the-art architectures.

Concept Breakdown

Selective SSM (S6)

A state space model where the parameters vary with the input, allowing the model to choose which information to keep or discard.

Parallel Scan

A technique to compute prefix sums or recurrences in parallel, adapted in Mamba for hardware-efficient SSM updates.

Linear Scaling

Computational complexity that grows proportionally to the sequence length ($$O(L)$$), enabling the processing of very long sequences.

Hardware-Awareness

Designing algorithms to minimize data movement between different levels of GPU memory (HBM vs. SRAM) to maximize throughput.

Discretization

The process of converting a continuous-time state space model into a discrete-time version for digital computation.

Mathematics

Continuous State Space Model

$$h'(t) = A h(t) + B x(t), \quad y(t) = C h(t)$$
The fundamental linear ODE that defines the evolution of the latent state $$h(t)$$ in a continuous SSM.
SymbolMeaning
$h(t)$Latent state at time $$t$$
$x(t)$Input signal at time $$t$$
$A, B, C$System matrices defining state transitions and output mapping

Selective Discretization

$$\bar{A} = \exp(\Delta A), \quad \bar{B} = (\Delta A)^{-1}(\exp(\Delta A) - I) \cdot \Delta B$$
In Mamba, the step size $$\Delta$$ and matrices $$B$$ and $$C$$ are functions of the input $$x$$, making the discretized matrices $$\bar{A}$$ and $$\bar{B}$$ time-varying.
