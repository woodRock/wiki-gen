"Attention Is All You Need" is the landmark 2017 paper by Vaswani et al. from Google Brain and Google Research that introduced the Transformer, a sequence transduction architecture based entirely on attention mechanisms, dispensing with recurrence and convolutions. The Transformer achieves state-of-the-art results on machine translation—28.4 BLEU on WMT 2014 English-German and 41.8 BLEU on WMT 2014 English-French—while being significantly more parallelizable and training in a fraction of the time of prior recurrent models. This architecture became the foundation of virtually all modern large language models, vision transformers, and multimodal AI systems.

Recurrence vs. Attention: The Parallelization Problem

Prior to the Transformer, dominant sequence models were based on recurrent neural networks (RNNs), particularly LSTMs and GRUs. Recurrent models compute hidden states sequentially: $$h_t = f(h_{t-1}, x_t)$$, where each step depends on the previous. This sequential dependency fundamentally limits parallelization during training—GPUs cannot process positions in parallel when each depends on the last. Figure 0

For long sequences, the sequential constraint becomes critical. Training on sequences of length $$n$$ requires $$n$$ serial steps regardless of batch size. Memory constraints further limit batching, making training on long sequences slow and expensive. RNNs also struggle to propagate information over very long distances due to vanishing and exploding gradients, though LSTMs partially address this.

Attention mechanisms had been used alongside recurrent networks in neural machine translation, allowing the decoder to "attend" to all encoder states at each decoding step rather than relying on a single context vector. These hybrid recurrent+attention models achieved strong results but retained the sequential computation bottleneck of the RNN.

The Transformer's radical departure was to eliminate recurrence entirely, relying solely on attention to model all dependencies between input and output positions. This enables full parallelization across all positions during training, reducing the number of sequential operations from $$O(n)$$ to $$O(1)$$, thereby transforming the scalability of sequence modeling.

Scaled Dot-Product Attention

The fundamental building block of the Transformer is scaled dot-product attention. Given queries $$Q \in \mathbb{R}^{n \times d_k}$$, keys $$K \in \mathbb{R}^{m \times d_k}$$, and values $$V \in \mathbb{R}^{m \times d_v}$$, attention computes: $$\text{Attention}(Q,K,V) = \text{softmax}(QK^T/\sqrt{d_k})V$$. The result is a weighted sum of values, where weights are determined by the compatibility (dot product) between each query and all keys. Figure 1

The scaling factor $$1/\sqrt{d_k}$$ is critical for numerical stability. Without it, for large $$d_k$$, the dot products $$QK^T$$ grow in magnitude, pushing the softmax into regions with very small gradients and causing vanishing gradient problems during training. Dividing by $$\sqrt{d_k}$$ keeps dot products in a range where the softmax function has meaningful gradients.

The softmax operation converts raw attention scores into a probability distribution over values, ensuring the output is a convex combination of value vectors. This normalization means each output position attends to a "probability-weighted mixture" of all input positions, with higher weight on more compatible key-query pairs.

The time complexity of scaled dot-product attention is $$O(n^2 \cdot d_k)$$ for sequence length $$n$$; the $$n^2$$ term comes from computing all $$n^2$$ query-key dot products. This quadratic scaling in sequence length is the primary computational limitation of the Transformer, motivating extensive subsequent work on efficient attention variants.

Multi-Head Attention

Multi-head attention extends scaled dot-product attention by running $$h$$ parallel attention functions on linearly projected versions of $$Q$$, $$K$$, and $$V$$. Each "head" uses different learnable projection matrices $$W_i^Q$$, $$W_i^K$$, and $$W_i^V$$ to project to lower dimensions $$d_k = d_v = d_{\text{model}}/h$$, computes attention independently, then concatenates and re-projects the results.

This parallel structure allows each head to attend to different representation subspaces and different positions simultaneously. In practice, different heads learn to attend to different linguistic or structural relationships: some heads capture syntactic dependencies, others semantic similarities, and others positional relationships. The multi-head design effectively provides the model with multiple "views" of the input at each attention layer.

The concatenation and final projection $$W^O$$ mixes information across heads, allowing the model to combine the diverse perspectives each head has captured. The total computation of multi-head attention is similar to single-head attention at the full dimension—the reduced per-head dimension compensates for the $$h$$-fold parallelism.

Multi-head attention also counteracts a limitation of single-head attention: averaging over softmax-weighted positions can wash out information about any single position. By running multiple independent attention functions, multi-head attention can simultaneously focus on multiple specific positions, preserving more information about individual token interactions.

Encoder-Decoder Architecture and Positional Encoding

The Transformer follows an encoder-decoder structure. The encoder, composed of $$N=6$$ identical layers each with multi-head self-attention and a position-wise feed-forward network, maps the input sequence to a sequence of continuous representations. The decoder, also comprising $$N=6$$ layers, generates the output sequence autoregressively using self-attention over previously generated tokens and cross-attention to the encoder output.

Each sub-layer employs residual connections ($$\text{output} = \text{LayerNorm}(x + \text{Sublayer}(x))$$) that facilitate the training of deep networks by providing gradient shortcuts. Layer normalization (as opposed to batch normalization) is used since sequence lengths vary across examples. The residual and LayerNorm structure became standard in virtually all subsequent deep learning architectures.

Since the Transformer has no inherent notion of sequence order—unlike RNNs, which process tokens sequentially—positional information must be injected explicitly. The paper uses sinusoidal positional encodings: $$\text{PE}_{(pos,2i)} = \sin(pos/10000^{2i/d_{\text{model}}})$$ and $$\text{PE}_{(pos,2i+1)} = \cos(pos/10000^{2i/d_{\text{model}}})$$. Sinusoids of different frequencies allow the model to attend by relative position: $$\text{PE}(pos+k)$$ is a linear transformation of $$\text{PE}(pos)$$, enabling easy generalization to sequence lengths not seen during training.

The decoder uses masked self-attention to prevent attending to future positions during training, which would constitute information leakage. The mask sets attention scores for future positions to $$-\infty$$ before softmax, ensuring each decoder position can only attend to earlier positions and itself.

Training, Regularization, and Impact

The Transformer was trained using the Adam optimizer with a custom learning rate schedule: warmup for the first 4000 steps followed by decay proportional to the inverse square root of the step number. This warmup prevents large gradient updates early in training when the model is poorly initialized. Dropout was applied to attention weights and residual connections with a rate of 0.1.

Label smoothing ($$\epsilon_{ls} = 0.1$$) was used during training: rather than targeting one-hot distributions, the model is trained to assign probability $$\epsilon_{ls}/V$$ to non-target tokens. This prevents the model from becoming overconfident and improves perplexity and BLEU scores, despite increasing raw training loss.

The Transformer achieved 28.4 BLEU on WMT 2014 English-German and 41.8 BLEU on English-French using a single model, surpassing all prior single-model and ensemble results. Training the "big" model took 3.5 days on 8 P100 GPUs—a fraction of the cost of comparable RNN models. These results were achieved in 2017; by 2024, Transformer-based models dominate virtually every language, vision, and multimodal benchmark.

The Transformer's impact extends far beyond machine translation. GPT, BERT, T5, and their descendants—the foundation of modern LLMs—are all direct extensions of this architecture. Vision Transformers (ViT) apply the same mechanism to image patches. The attention mechanism described in this paper forms the computational core of systems like GPT-4, Claude, and Gemini.

Concept Breakdown

Scaled Dot-Product Attention

The core attention function computing weighted sums of value vectors using query-key compatibility scores: $$\text{Attention}(Q,K,V) = \text{softmax}(QK^T/\sqrt{d_k})V$$. Scaling by $$1/\sqrt{d_k}$$ prevents vanishing gradients in the softmax when $$d_k$$ is large. This operation is the fundamental building block of all Transformer variants.

Multi-Head Attention

Parallel application of $$h$$ attention functions with different learned projections of queries, keys, and values. Each head computes attention in a $$d_{\text{model}}/h$$ dimensional subspace, capturing different relationships. Outputs are concatenated and projected, allowing the model to simultaneously attend to information from different representation subspaces.

Positional Encoding

Sinusoidal functions of position added to token embeddings to inject sequence order information. $$\text{PE}_{(pos,2i)} = \sin(pos/10000^{2i/d})$$ and $$\text{PE}_{(pos,2i+1)} = \cos(pos/10000^{2i/d})$$. The sinusoidal design allows the model to easily generalize to longer sequences via relative position relationships.

Residual Connections and Layer Normalization

Each sub-layer (attention or feed-forward) is wrapped with $$\text{LayerNorm}(x + \text{Sublayer}(x))$$. Residual connections enable gradient flow through deep networks, preventing vanishing gradients. Layer normalization normalizes across the feature dimension independently for each token, handling variable sequence lengths.

Masked Self-Attention

A self-attention variant used in the decoder that prevents attending to future positions by setting their attention scores to $$-\infty$$ before softmax. This ensures the model remains autoregressive during both training and inference, with each output token depending only on previously generated tokens.

Position-Wise Feed-Forward Network

A two-layer MLP applied identically and independently to each position in the sequence: $$\text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2$$ with $$d_{ff} = 2048$$ (four times the model dimension). This expands and contracts the representation, allowing per-position nonlinear transformations beyond what attention can provide.

Mathematics

Scaled dot-product attention

$$\text{Attention}(Q,K,V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$
Queries $$Q$$ and keys $$K$$ compute compatibility scores via dot products, scaled by $$1/\sqrt{d_k}$$ to control gradient flow. Softmax converts scores to attention weights (summing to 1 per query). The output is a weighted sum of value vectors $$V$$—each output position is an information-weighted mixture of all input positions.
SymbolMeaning
$Q$query matrix n x d_k
$K$key matrix m x d_k
$V$value matrix m x d_v
$d_k$key/query dimension, used as scaling factor
$\sqrt{d_k}$scaling to prevent dot products from growing too large

Multi-head attention

$$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1,\ldots,\text{head}_h)W^O$$
$$h$$ attention functions run in parallel, each on separately projected versions of $$Q, K, V$$ using learned matrices $$W_i^Q, W_i^K, W_i^V$$. Different heads capture different relationships (syntactic, semantic, positional). Concatenating and projecting combines all perspectives into a single representation.
SymbolMeaning
$h$number of attention heads (8 in base model)
$W^O$output projection matrix d_model x d_model
$\text{head}_i$Attention(QW_i^Q, KW_i^K, VW_i^V)

Sinusoidal positional encoding

$$\text{PE}_{(pos,2i)} = \sin\!\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right), \quad \text{PE}_{(pos,2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$
Sine and cosine functions at different frequencies encode position uniquely. Each dimension $$i$$ uses a different frequency, creating a unique fingerprint for each position. The sinusoidal design allows linear extrapolation to longer sequences: $$\text{PE}(pos+k)$$ is a linear function of $$\text{PE}(pos)$$, meaning relative positions are easy to compute from absolute encodings.
SymbolMeaning
$pos$token position in sequence
$i$dimension index
$d_{\text{model}}$model embedding dimension (512 in base model)

Position-wise feed-forward network

$$\text{FFN}(x) = \max(0,\; xW_1 + b_1)W_2 + b_2$$
A two-layer ReLU network applied identically to each position. Inner dimension $$d_{ff} = 2048$$ is $$4\times$$ larger than $$d_{model} = 512$$, providing a bottleneck expansion for nonlinear feature transformation. This layer is where most position-specific feature learning occurs, complementing the cross-position mixing of attention.
SymbolMeaning
$W_1$first weight matrix d_model x d_ff
$W_2$second weight matrix d_ff x d_model
$d_{ff}$inner dimension, 2048 in base model

Transformer learning rate schedule

$$lrate = d_{\text{model}}^{-0.5} \cdot \min\!\left(step^{-0.5},\; step \cdot warmup\_steps^{-1.5}\right)$$
Learning rate increases linearly during warmup ($$step < 4000$$) then decays as inverse square root. Warmup prevents large updates while model weights are poorly initialized. The $$d_{model}^{-0.5}$$ scaling ensures larger models use proportionally smaller learning rates. This schedule became standard for Transformer training.
SymbolMeaning
$step$current training step
$warmup\_steps$4000 in the paper
$d_{\text{model}}$model dimension affecting scale
